## Цель
Добавить возможность парсинга одного или нескольких телеграмм каналов за определенный промежуток времени и вывод полученной информации

## Этапы реализации

### Этап 1: Подготовка формы для сбора данных
1. **Изменение поля "Каналы"**: Заменить MultiSelect на Input компонент, который позволяет вводить названия каналов через запятую (например: `@ch1, @ch2` или `ch1, ch2`). Добавить валидацию формата каналов (удаление пробелов, проверка на пустые значения).
2. **Изменение поля "Временной промежуток"**: Заменить Select на DatePicker с возможностью выбирать начальную и конечную дату. Для MVP можно использовать нативный HTML5 date input или простую библиотеку. Добавить валидацию (начальная дата должна быть раньше конечной).

### Этап 2: Парсинг телеграмм каналов
1. **Создание модуля parsing на бэкенде**: 
   - Создать структуру `back/src/parsing/` с файлами: `service.ts`, `types.ts`
   - Создать функцию `parseChannels(channels: string[], startDate: Date, endDate: Date)` для парсинга постов
2. **Интеграция с Telegram API**:
   - Для MVP использовать заглушку (mock данные), так как реальная интеграция с Telegram API требует настройки Bot API токена и доступа к каналам
   - Добавить `AICODE-TODO` о необходимости интеграции с реальным Telegram API (Bot API или MTProto)
   - Структура ответа: массив постов с полями `channel`, `messageId`, `text`, `date`, `attachments` (опционально)
3. **Валидация входных данных**:
   - Проверка формата каналов (удаление @, проверка на пустые значения)
   - Проверка временного диапазона (не более 30 дней для MVP, чтобы избежать перегрузки)
4. **Обработка ошибок**:
   - Обработка случаев, когда канал недоступен или не существует
   - Обработка случаев, когда нет постов за указанный период
   - Обработка ошибок API (timeout, rate limit)

### Этап 3: API endpoint для парсинга
1. **Создание REST endpoint**: `POST /api/parse-channels`
   - Принимает: `{ channels: string[], startDate: string, endDate: string }`
   - Возвращает: `{ posts: Post[], totalCount: number, channels: string[] }`
   - Добавить базовую валидацию входных данных
2. **Интеграция с фронтендом**:
   - Добавить метод в `apiClient` для вызова endpoint
   - Добавить обработку ошибок на фронтенде

### Этап 4: Добавление функциональности на форму
1. **Обновление формы на главной странице**:
   - Заменить MultiSelect на Input для каналов
   - Заменить Select на DatePicker для временного промежутка
   - Добавить валидацию полей перед отправкой
2. **Интеграция с API**:
   - При заполнении полей и нажатии на кнопку "Запустить анализ" отправлять запрос на `/api/parse-channels`
   - Пока идет парсинг отображать LoadingSpinner
   - После завершения выводить результаты в консоль (console.log)
   - Добавить базовую обработку ошибок (вывод в консоль)

## Возможные риски

1. **Отсутствие реального Telegram API токена**: Для MVP используем заглушку, но это нужно явно отметить в коде
   - *Митигация*: Добавить `AICODE-TODO` и четко указать, что это mock данные

2. **Проблемы с DatePicker**: Нативный HTML5 date input может иметь проблемы с поддержкой в разных браузерах
   - *Митигация*: Для MVP использовать нативный input type="date", добавить `AICODE-TODO` о необходимости более продвинутого решения

3. **Валидация каналов**: Может быть сложно определить, существует ли канал без реального API
   - *Митигация*: Базовая валидация формата (удаление пробелов, проверка на пустые значения), реальная проверка будет добавлена при интеграции с Telegram API

4. **Производительность при большом количестве постов**: Парсинг большого количества каналов за длительный период может быть медленным
   - *Митигация*: Добавить ограничение на временной диапазон (30 дней для MVP), добавить `AICODE-TODO` о необходимости пагинации и асинхронной обработки

## Стратегия отката

1. **Если DatePicker вызывает проблемы**: Вернуться к Select с предустановленными опциями, добавить `AICODE-TODO`
2. **Если парсинг не работает**: Упростить до вывода mock данных в консоль, добавить `AICODE-TODO` о реальной интеграции
3. **Если валидация слишком сложная**: Упростить до базовой проверки на пустые значения

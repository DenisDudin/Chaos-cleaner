## Цель
Избавиться от библиотеки Tailwind CSS и переделать все компоненты с ее использованием на использование Shadcn/ui.

## Этапы реализации

### Этап 1: Установка и настройка Shadcn/ui
1. **Установка зависимостей**:
   - Установить `shadcn-ui` CLI и необходимые зависимости
   - Установить `class-variance-authority`, `clsx`, `tailwind-merge` (для утилит)
   - Установить `@radix-ui` компоненты, которые будут использоваться
   - Установить `lucide-react` для иконок (если нужно)

2. **Настройка проекта**:
   - Создать `components.json` для конфигурации Shadcn/ui
   - Настроить пути для компонентов
   - Настроить тему (темная тема с бирюзовым primary цветом)

3. **Удаление Tailwind**:
   - Удалить `tailwindcss` и `@tailwindcss/postcss` из `package.json`
   - Удалить `tailwind.config.js`
   - Удалить `postcss.config.js` или обновить его (убрать tailwind плагин)
   - Обновить `index.css` (убрать `@import "tailwindcss"`)

## Возможные риски

1. **Потеря кастомных стилей**: Shadcn/ui использует Tailwind под капотом, но мы хотим полностью убрать Tailwind
   - *Митигация*: Использовать CSS модули или inline стили для кастомизации, либо настроить Shadcn/ui без Tailwind (используя CSS переменные)

2. **Shadcn/ui требует Tailwind**: Shadcn/ui по умолчанию использует Tailwind CSS
   - *Митигация*: Использовать Shadcn/ui компоненты, но стилизовать их через CSS модули или CSS-in-JS. Либо использовать только Radix UI компоненты напрямую без Shadcn обертки.

3. **Потеря функциональности компонентов**: При миграции может быть потеряна какая-то функциональность
   - *Митигация*: Тщательно тестировать каждый компонент после миграции, сохранять все пропсы и поведение

4. **Размер бандла**: Shadcn/ui компоненты могут увеличить размер бандла
   - *Митигация*: Использовать tree-shaking, импортировать только нужные компоненты

## Стратегия отката

1. **Если миграция вызывает критические проблемы**: Вернуться к Tailwind, закоммитить изменения в отдельную ветку для дальнейшей работы
2. **Если Shadcn/ui не подходит**: Рассмотреть альтернативы (Radix UI напрямую, MUI, Chakra UI)
3. **Если потеря функциональности**: Восстановить функциональность через кастомные компоненты или патчи

## Примечания

- Shadcn/ui технически использует Tailwind CSS под капотом, но пользователь хочет полностью убрать Tailwind. Это означает, что нужно либо:
  1. Использовать только Radix UI компоненты напрямую и стилизовать их через CSS
  2. Использовать Shadcn/ui компоненты, но переписать их стили на чистый CSS
  3. Уточнить у пользователя, что Shadcn/ui требует Tailwind, и предложить альтернативу

- Для MVP лучше использовать вариант 2: взять структуру компонентов Shadcn/ui, но переписать стили на CSS модули или inline стили, чтобы полностью убрать зависимость от Tailwind.
